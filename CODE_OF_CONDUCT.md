# Code of Conduct

This project is meant to be direct, respectful, and useful.

## Expected Behavior

- Be respectful in disagreement.
- Critique ideas, code, and decisions without attacking people.
- Assume good faith, but accept correction when something is wrong.
- Prefer concrete bug reports, clear repro steps, and practical suggestions.

## Unacceptable Behavior

- Harassment, threats, or personal attacks
- Hate speech or discriminatory language
- Doxxing, credential sharing, or posting private infrastructure details
- Repeated bad-faith disruption in issues or pull requests

## Maintainer Responsibility

Maintainers may edit, hide, lock, or remove comments, issues, pull requests, and discussions that make the project harder to use or unsafe to participate in.

## Scope

This applies to repository issues, pull requests, releases, and any project spaces managed through GitHub.
